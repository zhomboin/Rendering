# Rendering 开发计划

日期：2026-03-21
项目：Rendering
文档状态：Active
文档角色：当前个人博客阶段的唯一中文总开发计划

## 计划目标

将 `Rendering` 打造成一个完成度更高的个人技术博客：具备稳定的发布基线、良好的阅读体验、可持续的本地 MDX 工作流、轻量分发能力，以及对未来 CMS 扩展友好的结构，但当前阶段不实际实现 CMS 功能。

## 当前状态

仓库已经具备：

- 公开首页、归档、文章、标签、About 页面
- 中文默认路由与英文镜像 UI 路由
- 本地 MDX 内容仓库
- 标签归档与标签筛选
- Pagefind 搜索集成
- `sitemap`、`robots`、JSON-LD
- 趣味黏土拟态视觉系统
- 图文模块、阅读进度与更完整的文章阅读体验

## 当前工作流

### 1. 发布基线收口

通过以下工作把当前站点收成更稳的发布状态：

- 同步主文档与仓库现状
- 增加明确的本地验证命令
- 增加 CI 核心检查
- 完善缺失页体验
- 补强关键交互的可访问性

### 2. 作者工作流与内容纪律

通过以下工作强化内容层：

- 明确 frontmatter 契约
- 增加发文模板与草稿生成脚本
- 增强内容校验
- 保持 repository helpers 作为公开内容的唯一来源

### 3. 分发能力补齐

在当前 SEO 基础上继续补齐：

- Feed 订阅能力
- manifest 支持
- 更完整的分享与分发基础

### 4. 精选内容迁移

从用户自己的 CSDN 原创博客中，迁移首批精选文章到本站，并以本站原生文章标准重新整理，而不是机械复制。

### 5. 未来 CMS 预留

保持架构对未来 CMS 阶段友好：

- 保留清晰的内容层边界
- 避免页面直接耦合文件解析细节
- 保持当前 UI 组件可复用

但当前阶段不提前加入登录、后台或数据库复杂度。

## 推荐执行顺序

1. 发布基线收口
2. 作者工作流与迁移模板
3. 分发能力补齐
4. 精选 CSDN 原创文章迁移
5. 再评估是否进入未来 CMS 阶段

## 当前阶段完成定义

当以下条件成立时，当前阶段就进入一个更成熟的状态：

- 本地验证与构建流程稳定可靠
- 产品文档与实现保持一致
- 新文章更容易以统一方式创建
- 归档内容在数量与技术深度上同步增长
- 没有任何当前阶段目标依赖 CMS 基础设施才能成立
