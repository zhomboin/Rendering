# Rendering 开发计划

日期：2026-03-19
项目：Rendering
文档状态：Active
文档角色：当前个人博客阶段的唯一中文总开发计划

## 计划目标

将 `Rendering` 交付为一个完整的个人技术博客，具备高质量公开阅读体验、本地 MDX 内容工作流、静态搜索与可扩展 SEO 基础能力；同时保持代码结构对未来 CMS 扩展友好，但当前阶段不实际实现 CMS 功能。

## 当前阶段边界

### 本计划涵盖

- 公开路由和读者侧 UI
- 本地内容架构
- 视觉系统与交互质量
- 搜索与内容发现
- SEO 基础能力
- 质量与部署准备

### 本计划不涵盖

- 登录与鉴权
- 后台管理面板
- 基于数据库的内容编辑
- 多作者发布
- 后端媒体管理

## 当前状态概览

仓库已经具备：

- 公开页面
- MDX 内容仓库层
- Pagefind 搜索集成
- 基础 metadata
- 趣味黏土拟态视觉系统
- 支持 figure / caption / gallery 的文章图文模块

## 核心工作流

### 1. 产品对齐与文档基线

保持 `docs/PRD.md`、`docs/development-plan.md`、中文主文档、`README.md`、`README.en.md` 和 `AGENT.md` 与真实产品和当前技术栈一致。

### 2. 公开博客体验完善

持续打磨：

- 首页叙事和层级
- 归档页扫描效率
- 文章页阅读舒适度
- 标签驱动浏览
- 响应式体验
- 图文文章阅读体验

### 3. 内容系统与编辑工作流

保持：

- 本地 MDX 为当前主写作流程
- 稳定的 frontmatter 契约
- 可靠的标签聚合
- 仅通过 repository helpers 获取公开内容
- 内容层行为变更补测试

### 4. 搜索与 SEO

继续保持搜索稳定，并补齐：

- 路由 metadata
- 文章 metadata
- `sitemap.xml`
- `robots.txt`
- JSON-LD

### 5. 质量、性能与部署准备

保持：

- helper 与内容层测试通过
- 生产构建和搜索索引构建成功
- 移动端和桌面端阅读体验可靠
- focus 状态和 reduced-motion 支持稳定

### 6. 未来 CMS 扩展准备

当前允许的准备工作：

- 将内容访问统一封装在 repository helpers 后
- 避免页面直接耦合文件解析细节
- 为未来替换内容源保留结构空间
- 让 UI 组件在未来后台阶段仍可复用

当前不允许做的事：

- 为“也许以后会用”而提前做登录流程
- 在当前没有明确需求时加入数据库依赖
- 引入并不真正需要的 admin 路由
- 按未来 dashboard 的逻辑来反向塑造当前公开博客

## 推荐优先级顺序

1. 先稳定 PRD 与计划基线
2. 持续打磨公开阅读与浏览体验
3. 强化内容质量与作者工作流纪律
4. 在有明确价值时补强 SEO
5. 继续收紧 QA 和部署准备
6. 最后再定义第一个 CMS 扩展阶段

## 当前阶段完成定义

当前阶段满足以下条件时，可以认为基本完成：

- 公开博客整体足够完整、连贯、成熟
- MDX 工作流稳定且易用
- 搜索与 metadata 可靠
- 项目易于维护与部署
- 当前阶段没有任何必须依赖 CMS 才能成立的关键缺口
