# Rendering 产品需求文档

日期：2026-03-19
项目：Rendering
文档状态：Active
文档角色：当前个人博客阶段的唯一中文 PRD

## 产品定义

`Rendering` 在当前阶段是一个个人技术博客，而不是 CMS 产品。

这一阶段的产品是一套面向公开访问的发布型站点，用于承载关于前端系统、渲染、交互行为、动效、性能以及工程思考的长文写作。它应该体现明确的作者表达、编辑气质、鲜明的视觉个性，以及高质量的阅读体验。

当前优先级：

- 优秀的公开阅读体验
- 清晰的个人编辑气质和写作人格
- 稳定的本地 MDX 工作流
- 面向已发布文章的搜索与 SEO
- 即使没有后台也显得完整的博客前台

## 当前阶段目标

交付一个完整的个人博客体验，至少支持：

- 使用本地 MDX 文件发布技术文章
- 通过首页、归档页、标签页和文章页浏览内容
- 通过静态全文搜索发现内容
- 提供可扩展的 SEO 基础能力
- 建立一套围绕趣味黏土拟态、鲜艳配色与长文阅读体验的稳定视觉系统

## 当前范围内功能

- 首页、博客归档、文章详情、标签页、About 页面
- `content/posts` 下的本地 MDX 内容
- frontmatter 元数据、草稿过滤、标签聚合
- 上一篇 / 下一篇文章导航
- `figure / caption / gallery` 图文模块
- Pagefind 搜索与标签驱动浏览
- 路由 metadata、canonical URL、Open Graph、Twitter metadata
- 默认亮色主题、暗色主题支持
- 文章阅读进度跟踪

## 当前明确不做

- 登录与作者鉴权
- 管理后台
- 在线编辑器
- 媒体库
- 角色与权限系统
- 基于数据库的文章管理
- 定时发布工作流
- 评论审核后台
- 数据分析后台
- 多作者发布

## 用户体验要求

- 首页应像编辑入口页，而不是营销 landing page。
- 归档页应支持快速扫描，并具备 featured 层级和明确元信息。
- 文章页是最高优先级体验，必须保证阅读舒适度、图文混排质量、代码块表现和低干扰动效。
- 搜索应像编辑型内容产品的自然延伸，而不是通用启动器。

## 视觉方向

当前视觉方向为：趣味黏土拟态个人博客。

关键特征：

- 默认亮色主题，暗色主题保持同一人格
- 圆润、柔软、可触摸的卡片与胶囊控件
- 珊瑚橙、明黄、湖蓝、草绿、紫色、亮粉等鲜艳但受控的强调色
- 黏土拟态感来自体块、阴影和层级，而不是堆砌特效
- 首页与导航更活泼，文章页更安静、更偏编辑型
- 图文文章中的图片区域清晰、柔和、留白充足

应避免：

- 回到冷峻 terminal / cyber 控制台风格
- 泛紫色 AI 渐变套版
- 过强 bloom 和 blur
- 用装饰性动画与正文争抢注意力
- 把正文做成过度糖果化的玩具区域

## 技术方向

- Next.js App Router
- React
- TypeScript
- 本地 MDX 内容
- `gray-matter`
- `@mdx-js/mdx`
- `remark-gfm`
- Pagefind

推荐架构：

- 基于本地文件的内容仓库层
- 静态友好的路由方式
- 基于路由的 metadata 生成
- 可复用 UI 组件
- 当前阶段不依赖数据库

## 成功标准

当前阶段满足以下条件时，即可认为成功：

- 站点在没有后台管理能力的情况下，依然作为个人博客成立
- 通过本地 MDX 发布新文章足够直接
- 所有公开页面视觉上一致、完整
- 文章页能同时承载高质量长文和图片内容
- 搜索和标签导航可以有效帮助读者发现内容
- 站点具备扎实的 SEO 基础和后续扩展空间
