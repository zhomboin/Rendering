# Rendering 产品需求文档

日期：2026-03-23
项目：Rendering
文档状态：Implemented
文档角色：当前个人博客实现阶段的最终中文 PRD

## 产品定义

`Rendering` 是一个个人技术博客，而不是 CMS 产品。

当前已经完成实现的产品，是一套面向公开访问的发布型站点，围绕以下目标建立：

- 长篇技术写作
- 带有趣味黏土拟态气质的编辑型视觉风格
- 默认亮色、支持暗色切换的主题系统
- 基于本地 MDX 的写作流程
- 适合文字、代码和图片混排的阅读体验
- 静态搜索、SEO 与分发基础设施

## 已交付范围

读者现在可以：

- 从首页、博客归档、博客页内标签筛选和搜索中发现内容
- 舒适地阅读图文混排技术文章
- 在中文默认路由与英文镜像 UI 之间切换
- 通过标签与上一篇 / 下一篇导航继续浏览内容

作者现在可以：

- 通过本地 MDX 文件发布文章
- 使用 frontmatter 管理元数据
- 在不引入 CMS 的前提下持续扩展内容归档
- 在发布前完成本地校验与构建验证

## 已实现功能

- 首页、博客归档、文章详情、About 页面
- 中文默认路由与 `/en` 英文镜像站点 UI
- `content/posts` 下的本地 MDX 内容源
- frontmatter 驱动的文章元数据
- 基于 `/blog?tag=...` 的博客页标签筛选
- Pagefind 搜索
- route metadata、`sitemap.xml`、`robots.txt`、`feed.xml`、`manifest.webmanifest`、JSON-LD
- 阅读进度
- `figure / caption / gallery / lightbox` 图文模块
- 动态 OG 图生成
- 明暗主题切换
- 基于 GitHub Actions 的验证流程

## 明确不做

- CMS 实现
- 登录与鉴权
- 后台路由或管理面板
- 数据库驱动的内容编辑
- 评论与审核
- 分析后台
- Newsletter 系统
- 多作者工作流

这些能力只保留为未来扩展方向，不属于当前已交付产品。

## 当前产品气质

当前博客应当体现以下特征：

- 更像编辑型个人博客，而不是模板站
- 外层更有趣味，正文更安静耐读
- 色彩鲜艳，但不做高噪声表达
- 适合图片与代码出现，但不把正文做成装饰性画廊
- 技术内容扎实，适合真实工程阅读

## 当前完成度结论

当前个人博客实现阶段可以认为已经完成，因为：

- 站点在没有 CMS 的前提下已经完整成立
- MDX 发文流程稳定可重复
- 搜索、SEO、Feed、Manifest 已经接齐
- 本地验证与 GitHub CI 已经稳定
- 内容归档已经具备公开文章与原创迁移文章

## 正式上线前仍需处理的事项

剩余事项属于运营与发布层，而不是产品定义缺口：

- 正式域名与部署平台
- 最终作者身份与站点公开文案
- 首批上线文章的最终确认
- 正式发布动作本身

## 后续方向

如果未来继续扩展，应视为新的阶段，单独立项并拆分 specs / plans，例如：

- CMS 写作后台
- 管理端功能
- 登录与鉴权
- 多作者协作
- 更完整的数据分析与运营工具
