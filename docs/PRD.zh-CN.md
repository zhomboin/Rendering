# Rendering 产品需求文档

日期：2026-03-21
项目：Rendering
文档状态：Active
文档角色：当前个人博客阶段的唯一中文 PRD

## 产品定义

`Rendering` 当前是一个个人技术博客，而不是 CMS 产品。

它是一套面向公开访问的发布型站点，围绕以下目标展开：

- 长篇技术写作
- 具有趣味黏土拟态气质的编辑型视觉风格
- 默认亮色、支持暗色切换的主题系统
- 基于本地 MDX 的写作流程
- 适合文字、代码和图片混排的阅读体验
- 静态搜索与 SEO 基础能力

## 当前阶段目标

当前阶段要交付的是一个可以独立成立的个人博客，而不是后台系统的展示外壳。

读者应当可以：

- 从首页、归档、标签和搜索中发现内容
- 舒适地阅读图文混排技术文章
- 在中文默认站点与英文镜像 UI 之间切换
- 通过标签归档和相关文章持续浏览内容

作者应当可以：

- 通过本地 MDX 持续发布文章
- 使用 frontmatter 管理文章元数据
- 在不引入 CMS 的前提下持续扩展站点内容

## 当前范围

- 公开路由与读者端 UI
- 中文默认路由与 `/en` 英文镜像站点 UI
- 本地 MDX 内容源
- frontmatter 驱动的元数据
- 标签归档与标签筛选
- Pagefind 搜索
- route metadata、`sitemap.xml`、`robots.txt`、JSON-LD
- 阅读进度跟踪
- `figure / caption / gallery / lightbox` 图文模块
- 明暗主题切换

## 当前明确不做

- CMS 实现
- 登录与鉴权
- 后台管理路由
- 数据库驱动的内容编辑
- 评论与评论审核
- 数据分析后台
- Newsletter 系统
- 多作者工作流

这些能力只保留为未来阶段扩展，不进入当前产品定义。

## 当前产品气质

这个博客应该同时具备以下特征：

- 更像编辑型个人博客，而不是模板站
- 外层更有趣味，正文更安静耐读
- 色彩鲜艳但不过度喧闹
- 适合图片与代码出现，但不把正文做成装饰画廊
- 技术内容足够扎实，适合真实工程阅读

## 当前实现现状

仓库目前已经具备：

- 首页、归档、文章页、标签页、标签归档页、About 页
- 中文默认与英文镜像路由结构
- `content/posts` 下的本地 MDX 内容
- 标签归档与标签筛选浏览
- Pagefind 搜索弹层
- `sitemap`、`robots`、结构化数据
- 阅读进度与文章目录体验
- 图文模块与 lightbox
- 趣味黏土拟态视觉系统与主题切换

## 下一阶段优先事项

下一轮当前阶段开发应优先完成：

1. 发布基线收口
2. 作者工作流与迁移模板增强
3. Feed / manifest 等分发能力补齐
4. 从 CSDN 精选迁移原创文章到本站

## 内容方向

下一阶段内容增长应优先围绕以下主线：

- 数据库
- 系统
- 服务器
- 实战型工程笔记

首批迁移文章采用“人工精选 + 明显改写 + 统一风格整理”的方式，而不是批量导入。

## 成功标准

当前阶段可以认为成功，当：

- 站点在没有 CMS 的前提下依然完整成立
- MDX 发布流程保持稳定可持续
- 搜索与 SEO 足够可靠
- 内容归档逐渐变得更厚、更有技术深度
- 迁入本站的原创文章能自然融入当前博客风格
